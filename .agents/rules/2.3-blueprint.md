# 2.3 вЂ” Project Blueprint

## Purpose

Define repo structure, folder layouts, and file placement rules. This ensures every project is navigable and consistent, whether it's a simple frontend or a full monorepo.

## Project Profiles (ABSOLUTE)

Every project MUST declare one profile:

| Profile | Description | Example |
|---------|-------------|---------|
| **F** | Frontend Only | Next.js site, Vite SPA |
| **B** | Backend Only (API) | Fastify REST API |
| **FB** | Frontend + Backend | Web app with custom API |
| **M** | Mobile Only (Expo) | Mobile app |
| **MB** | Mobile + Backend | Mobile app with custom Fastify API |
| **FBM** | Fullstack + Mobile (Monorepo) | Complete product |
| **SF** | Frontend + Supabase | Web app with Supabase (no custom backend) |
| **SM** | Mobile + Supabase | Mobile app with Supabase (no custom backend) |

Profile MUST be declared in `PROJECT.yaml`:

```yaml
profile: "FB"
```

## Backend Module Structure (AUTHORITATIVE)

Every backend module under `modules/` MUST follow:

```
modules/[name]/
в”њв”Ђв”Ђ [name].controller.ts    # route handler (thin вЂ” delegates to service)
в”њв”Ђв”Ђ [name].service.ts       # business logic
в”њв”Ђв”Ђ [name].repository.ts    # database queries
в”њв”Ђв”Ђ [name].schema.ts        # Zod schemas (request/response DTOs)
в”њв”Ђв”Ђ [name].types.ts         # TypeScript types
в””в”Ђв”Ђ [name].test.ts          # tests
```

- Controllers: thin, validate input, call service, return response
- Services: business logic, orchestrate repositories
- Repositories: database access ONLY (Prisma queries)
- DB access in controllers FORBIDDEN
- Business logic in repositories FORBIDDEN

## File Naming (ABSOLUTE)

- All files and directories: `kebab-case`
- React components: `kebab-case.tsx` (e.g., `user-profile.tsx`)
- Component export: `PascalCase` (e.g., `export function UserProfile()`)
- Tests: `[name].test.ts` or `[name].spec.ts`
- Schemas: `[name].schema.ts`
- Types: `[name].types.ts`

## Monorepo Package Rules (AUTHORITATIVE)

- pnpm workspaces REQUIRED for monorepos
- Every package MUST have `src/index.ts` as public entry point
- Packages MUST NOT load dotenv
- Packages MUST NOT depend on `process.env` at import time
- Cross-app imports FORBIDDEN вЂ” use packages
- `@pkg/[name]` path aliasing REQUIRED for package imports

## Root Scripts (AUTHORITATIVE)

Root `package.json` MUST include:

```json
{
  "scripts": {
    "dev": "...",
    "build": "...",
    "lint": "...",
    "typecheck": "...",
    "test": "..."
  }
}
```

### [STANDARD] For Monorepos, add per-app shortcuts:

```json
{
  "scripts": {
    "api": "pnpm --filter api dev",
    "web": "pnpm --filter web dev",
    "mobile": "pnpm --filter mobile start"
  }
}
```

### [STANDARD] For Docker-based projects, add:

```json
{
  "scripts": {
    "docker:up": "...",
    "docker:down": "...",
    "docker:logs": "...",
    "docker:build": "..."
  }
}
```
